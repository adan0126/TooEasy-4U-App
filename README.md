Problematica:
En México, millones de personas toman decisiones financieras sin contar con los conocimientos básicos necesarios para administrar su dinero de forma responsable. Esta situación impacta directamente su estabilidad económica, bienestar emocional y calidad de vida. TooEasy surge como una respuesta digital a esta problemática, enfocada en aprender haciendo.

Aquí se llevara acabo el desarrollo de la aplicación móvil

Esperar indicaciones, si no estan seguro no hacer nada.

Reglas del negocio:
1 Se crea una nueva branch para cada nueva funcionalidad o corrección de errores.
2 Cada branch debe ser revisada por al menos un compañero antes de hacer merge a main
3 El html, css y js deben estar dentro de una carpeta que haga mención al modulo que se trabaja.
4 Los commits deben ser claros y descriptivos.
5 No hacer push directo a main.
6 Mantener la documentación actualizada con cada cambio realizado.
7 Realizar pruebas exhaustivas antes de desplegar cualquier cambio en producción.
8 Respetar los plazos establecidos para cada tarea.
9 Comunicar cualquier problema o retraso al equipo de inmediato.
10 Mantener una comunicación abierta y constante con el equipo.
11 Respetar las decisiones tomadas por el equipo y la dirección del proyecto.
12 Priorizar la calidad del código sobre la cantidad de funcionalidades implementadas
